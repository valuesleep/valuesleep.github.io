文件上传原理

在文件上传的功能处，若服务端脚本语言未对上传的文件进行严格验证和过滤，导致恶意用户上传恶意的脚本文件时，就有可能获取执行服务端命令的能力，这就是文件上传漏洞。

文件上传漏洞高危触发点

相册 头像上传 视频 照片分享
附件上传（论坛发帖、邮箱）
文件管理器
注：如果移动端也存在类似的操作的话，那么相同的原理，也存在文件上传漏洞的风险。

less5

仔细观察发现.ini被放出来 且提示有readme.php
`user.ini ： 自 PHP 5.3.0 起，PHP 支持基于每个目录的 .htaccess 风格的 INI 文件。此类文件仅被
CGI／FastCGI SAPI 处理。此功能使得 PECL 的 htscanner 扩展作废。如果使用 Apache，则用
.htaccess 文件有同样效果。

除了主 php.ini 之外，PHP 还会在每个目录下扫描 INI 文件，从被执行的 PHP 文件所在目录开始一直上升到 web
根目录（$_SERVER[‘DOCUMENT_ROOT’] 所指定的）。如果被执行的 PHP 文件在 web 根目录之外，则只扫描该目录。

在 .user.ini 风格的 INI 文件中只有具有 PHP_INI_PERDIR 和 PHP_INI_USER 模式的 INI
设置可被识别。

两个新的 INI 指令，user_ini.filename 和 user_ini.cache_ttl 控制着用户 INI 文件的使用。

user_ini.filename 设定了 PHP 会在每个目录下搜寻的文件名；如果设定为空字符串则 PHP 不会搜寻。默认值是
.user.ini。

user_ini.cache_ttl 控制着重新读取用户 INI 文件的间隔时间。默认是 300 秒（5 分钟）。
`

以上重要的是
PHP 还会在每个目录下扫描 INI 文件，从被执行的 PHP 文件所在目录开始一直上升到 web

依次上传user.ini test.txt
访问readme.php

less6

对比less4发现源码少了行
$file_ext = strtolower($file_ext); //转换为小写
所以把文件后缀改成类似.Php且没被过滤的上传

less7

缺少$file_ext = trim($file_ext); //收尾去空
构造‘.php ’（注意后缀有空格）

less8

对比less4源码发现少了行$file_name = deldot($file_name);//删除文件名末尾的点
那么直接把.php改写成.php.(windows的特性) 会把.php.识别成.php
由于主机是windows直接对文件重命名会被直接更改为.php
可以用burpsuit抓包更改
两个点不行的话多加几个点

less9

对比前几关少了$file_ext = str_ireplace(‘::$DATA’, ‘’, $file_ext);//去除字符串::$DATA
Windows系统会把::$DATA当作文件流来进行处理，在文件名后面添加::$DATA可以绕过上述检测代码的查验

我们在得到该文件的地址后，去掉后面的::$DATA，发现该文件已经被解析成功

less10

过滤的顺序是先删除文件最后的空格，再上传文件末尾的点，将文件后缀名提取并全部转化成小写，然后删除::$DATA字符串，最后再次删除文件末尾的空格
构造后缀.php. .
当该文件传输到目标文件夹后，是以.php.为后缀 由于Windows系统的特性，会自动删除后面的点变成.php文件
